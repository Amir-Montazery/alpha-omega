About

The Alpha-Omega Assurance Assertions project is an attempt to provide an important missing piece of the open source security puzzle: a way to communicate about the security posture of open source software. It supplements other efforts, including OpenSSF Scorecards, static analysis using tools like CodeQL and Snyk Code, dependency analysis leveraging GitHub Security Advisories, and direct security reviews.

Assurance Assertions provides consumers (developers, compliance teams, security assurance, etc.) a way to measure the security quality of a project and evaluate policies to understand whether those projects meet their expectations.

Imagine being able to describe a policy like, "The only projects that implement cryptography should be OpenSSL, BoringSSL, or LibreSSL". Assurance Assertions can enable exactly this.

The best part: Assurance Assertions is open source. You use the public instance (this web-site you're reading right now) or you can run your own instance privately. You can generate assertions (or create your own), evaluate policies (or create your own), and use the results to make decisions within the confines of your own organization. Of course, we'd greatly appreciate you contributing back to help make the project more successful.

Assurance Assertions is part of the OpenSSF Alpha-Omega project.
